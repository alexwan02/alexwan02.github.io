Transmitting Network Data Using Volley[译]

发表于 2015-08-15   |  

Volley 概述

Volley是开源的Android网络请求库。可以让app网络请求更加快捷，更重要的是让网络请求更迅速。
Volley的特点

1.自动执行网络请求
2.多线程并发的网络连接
3.使用标准Http缓存协议来构建磁盘和内存缓存
4.支持请求的优先排序
5.支持取消一个或多个请求
6.请求重试和补偿机制
7.异步处理返回结果，正确更新UI
8.调试和追踪工具

Volley善于在远程调用操作来与UI交互，如获取结构化数据的搜索结果页。Volley可以和任何协议轻松集成。支持
String，images，和JSON等类型数据，Volley包种内置了你需要的功能，只需要关心app的功能逻辑即可。
Volley在内存来保留所有的响应，因此Volley不适合用来出来大的下载和数据流操作。文件下载操作建议考虑使用DownloadManager
Google的Volley的核心库位于frameworks/volley下，包括主要的请求分发逻辑代码。volley/toolbox包括
一套常用的实用工具。除了Google官方的开源包，Github上
也提供了Volley的相关源码。

（一）Volley库的引入：

（1）通过引入库的方式
1.使用Git命令克隆Volley库（本文使用github的）

git clone https://github.com/mcxiaoke/android-volley

2.导入下载库，在Android项目引入Volley作为项目的依赖库
（2）Gradle配置的方式
1.在Gradle配置文件中加入

compile 'com.mcxiaoke.volley:library:1.0.18'

（二）发送简单的请求

使用Volley对象高效率创建RequestQueue对象，传入请求Request对象。RequestQueue主要用于管理网络操作，
读写缓存，数据解析等线程的处理。Request处理请求响应的数据解析任务，Volley负责分发数据解析处理的结果到主
线程。
本节讲解如何使用Volley.newRequestQueue方法创建RequestQueue对象来请求网络数据
1.在AndroidManifest中添加网络请求权限

<uses-permission android:name="android.permission.INTERNET"/>

2.创建RequestQueue

final TextView mTextView = (TextView) findViewById(R.id.text);
...
// 初始化
RequestQueue queue = Volley.newRequestQueue(this);
String url = "http://www.baidu.com";
// 获取StringResponse
StringResponse stringResponse = new StringRequest(Request.Method.GET, url 
        new Response.Listener<String>(){
            @Override
            public void onResponse(String response){
                // 显示返回的String的前500个字符
                mTextView.setText("Response is :" + response.subString(0 , 500));
            }
        },new Response.ErroListener(){
            @Override
            public void onErrorResponse(VolleyError error){
                mTextView.setText("That didn't work !");
            }
        });
// 添加请求带队列中
queue.add(stringRequest);

Volley会传递返回到的解析后的响应到主线程中，可以根据返回的响应来控制UI的，除此Volley还提供了可以
取消请求任务的相关方法。

一、高效加载大图

图形有不同的形状与大小。在大多数情况下它们的实际大小都比需要呈现的尺寸大很多。系统的图库应用显示那些我们使用相机拍摄的照片，但是那些图片的分辨率通常都比设备屏幕的分辨率要高很多。
考虑到应用是在有限的内存下工作，我们只需要在内存中加载一个低分辨率的照片即可。为了更便于显示，这个分辨率的照片应该是与其对应的UI控件大小相适配。加载超过屏幕分辨率的高分辨率照片不仅对显示没有任何的提升，还会占用内存资源，此外在快速滑动图片时容易产生额外的效率问题。
