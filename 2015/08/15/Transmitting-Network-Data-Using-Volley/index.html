Android：网络请求框架Volley[译]

发表于 2015-08-15   |  

Volley是开源的Android网络请求库。可以让app网络请求更加快捷，更重要的是让网络请求更迅速，具有以下几个方面的特点：

1.自动执行网络请求
2.多线程并发的网络连接
3.使用标准Http缓存协议来构建磁盘和内存缓存
4.支持请求的优先排序
5.支持取消一个或多个请求
6.请求重试和补偿机制
7.异步处理返回结果，正确更新UI
8.调试和追踪工具

Volley善于在远程调用操作来与UI交互，如获取结构化数据的搜索结果页。Volley可以和任何协议轻松集成。支持String，images，和JSON等类型数据，Volley包种内置了你需要的功能，只需要关心app的功能逻辑即可。Volley在内存来保留所有的响应，因此Volley不适合用来出来大的下载和数据流操作。文件下载操作建议考虑使用DownloadManager
Google的Volley的核心库位于frameworks/volley下，包括主要的请求分发逻辑代码。volley/toolbox包括一套常用的实用工具。除了Google官方的开源包，Github上也提供了Volley的相关源码。

一、Volley库的引入

1.通过引入库的方式
（1）使用Git命令克隆Volley库（本文使用github的）

git clone https://github.com/mcxiaoke/android-volley

（2）导入下载库，在Android项目引入Volley作为项目的依赖库
2.Gradle配置的方式
（1）在Gradle配置文件中加入

compile 'com.mcxiaoke.volley:library:1.0.18'
二、发送简单的请求

使用Volley对象高效率创建RequestQueue对象，传入请求Request对象。RequestQueue主要用于管理网络操作，读写缓存，数据解析等线程的处理。Request处理请求响应的数据解析任务，Volley负责分发数据解析处理的结果到主线程。本节讲解如何使用Volley.newRequestQueue方法创建RequestQueue对象来请求网络数据

1、在AndroidManifest中添加网络请求权限
<uses-permission android:name="android.permission.INTERNET"/>
2、创建RequestQueue
final TextView mTextView = (TextView) findViewById(R.id.text);
...
// 初始化
RequestQueue queue = Volley.newRequestQueue(this);
String url = "http://www.baidu.com";
// 获取StringResponse
StringResponse stringResponse = new StringRequest(Request.Method.GET, url 
        new Response.Listener<String>(){
            @Override
            public void onResponse(String response){
                // 显示返回的String的前500个字符
                mTextView.setText("Response is :" + response.subString(0 , 500));
            }
        },new Response.ErroListener(){
            @Override
            public void onErrorResponse(VolleyError error){
                mTextView.setText("That didn't work !");
            }
        });
// 添加请求带队列中
queue.add(stringRequest);

Volley会传递返回到的解析后的响应到主线程中，可以根据返回的响应来控制UI的，除此Volley还提供了可以取消请求任务的相关方法。

3、Volley请求的过程。

发送一个请求，只需要创建一个Request对象，使用RequestQueue的add()方法添加到队列中。请求的数据通过传输通道获取服务和数据响应。
当你调用add()方法的时候，Volley会运行一个缓存线程和一个网络分发的线程池。添加请求到队列后，先通过缓存线程判断当前请求是否可以使用缓存来返回数据。如果有缓存则返回缓存响应和解析的缓存数据。如果没有获取到缓存响应，则将请求加入到网络请求队列中。第一个可用的网络线程从队列中获取到请求，建立http传输通道，解析线程返回的响应，如果可以使用缓存的话，同时将解析的数据保存到缓存中，最后将解析的响应数据传递给主线程。

注意：耗时操作如阻塞的I/O操作和数据解析/解码操作都在线程中处理，你可以从任何线程中添加请求，但是响应的数据总是在主线程中传输。

4、取消请求

使用Request对象调用cancel()方法来取消你的请求。一旦请求被取消，Volley能够确保你的响应相关处理不会被调用，也就是说你在Activity的onStop()方法中取消所有的请求，而不用在你的handler中去判断getActivity() == null，无论onSaveInstanceState()方法是否已经被调用，或者其他种情况。
为了利用这个特点，你通常只要通过检测工作中的请求来在需要的时候来取消他们。还有个更简单的方法：你可以为每个请求配置tag属性，用这个tag属性来取消你要的请求。比如：你可以为给Activity的请求添加Tag，在onStop()方法中调用requestQueue.cancellAll(this)来取消相关Activity中的数据请求。相似：可以在Viewpager中为所有的图片请求根据各自的tab添加对应的tag属性，在切换页面的时候取消请求，但要保证取消的请求不被其他tab页使用。
下面是一些对应的代码
（1）在你的请求中定义tag值

public static final String TAG = "MyTag";
StringRequest stringRequest;// 假设存在
RequestQueue requestQueue; // 假设存在
// 设置tag
stringRequest.setTag(TAG);
// 添加请求到队列中
requestQueue.add(stringRequest);

（2）在Activity的onStop()方法中取消所有对应tag属性的请求。

@Override
protected void onStop(){
    super.onStop();
    if(requestQueue != null){
        requestQueue.cancellAll(TAG);
    }
}

小心应用取消请求的操作，如果你依赖响应处理来改变对应状态或关闭其他操作，要做出对应的说明。

三、创建请求队列

Volley使用Volley.newRequestQueue来创建一个RequestQueue对象，执行请求操作。创建RequestQueue对象需要传入缓存和网络对象，为了减少内存开支在创建对象的时候可以考虑使用单例设计模式来管理RequestQueue对象

1、创建缓存和网络对象

RequestQueue需要两个对象来执行任务：一个网络对象来执行请求传输，一个缓存对象来缓存请求的数据。Volley提供了默认的缓存类DiskBaseCache和网络具体实现类BasicNetWork。BasicNetWork是Volley默认的网络实现，初始化建立网络连击的Client对象，根据android版本号，创建不同的Client对象AndroidHttpClient-基于Apache的HttpClient的具体实现和HttpURLConnection-Android提供的建立网络连接的对象

当android api版本小于9（Gingerbread）时，使用AndroidHttpClient。高于9时使用HttpURLConnection。更多介绍查看Android’s HTTP Clients

创建运行在所有版本的app时，可以通过检查Android设备的版本号来选择HttpClient

HttpStack stack;
...
// 如果设备运行的android版本大于9(Gingerbread)
if(Build.VERSION.SDK_INT >= Build.VERSION_CODES.GINGERBREAD){
    // 使用HttpURLConnection
}else{
    // 使用AndroidHttpClient
}
Network netWork = new BasicNetwork(stack);

如果仅仅是请求一次，不想加入线程池，可以在需要的地方创建RequestQueue，然后在响应或错误返回中调用stop方法。常用的用法是：创建一个单例模式的RequestQueue对象，在app生命周期内，只需要实例化一次。

2、使用单例设计模式

如果app频繁请求网络数据，使用单例模式来创建RquestQueue效率更高。实现单例模式有多种方式，建议是定义单例模式的类，封装RequestQueue对象和其他Volley的功能。另外一种方式是创建一个Application的子类，在Application.onCreate()方法中创建一个静态的RequestQueue对象，这个方式不推荐使用，因为使用一个静态单例模式更加模块化。
RequestQueue强烈建议使用Application的context初始化，而不是使用Activity的context参数，不必在每次创建Actvitiy时都要进行初始化，保证高效使用单例模式。
下面是封装了RequestQueue和ImageLoader的一个单例模式

public class MySingleton{
    private static MySingleton mInstance;
    private RequestQueue mRequestQueue;
    private ImageLoader mImageLoader;
    private static Context mCtx;
    private MySingleton(Context context){
        mCtx = context;
        mRequestQueue = getRequestQueue();
        mImageLoader = new ImageLoader(mRequestQueue,new ImageLoader.ImageCache(){
        private final LruCache<String,Bitmap> cache = new LruCache<String,Bitmap>(20);
            @Override
            public Bitmap getBitmap(String url){
                return cache.get(url);
            }
            @Override
            public void putBitmap(String url , Bitmap bitmap){
                cache.put(url,bitmap);
            }
        })
    }
    public static syncronized MySingleton getInstance(Context context){
        if(mInstance == null){
            mInstance  = new MySingleton(context);
        }
        return mInstance;
    }
    public RequestQueue getRequestQueue(){
        if(mRequestQueue == null){
            // 使用 getApplicationContext()作为键值
            mRequestQueue = Volley,newRequestQueue(mCtx.getApplicationContext());
        }
        return mRequestQueue;
    }
    public <T> void addToRequestQueue(Request<T> req){
        getRequestQueue.add(req);
    }
    public ImageLoader getImageLoader(){
        return mImageLoader;
    }
}

使用单例

// 获取RequestQueue
RequestQueue queue = MySingleton.getInstance(this.getApplicationContext());
getRequestQueue();
...
// 添加请求到队列中
MySingleton.getInstance(this).addToRequestQueue(stringRequest);
四、创建请求对象

Volley内置了一下几个常用的请求类型

1.StringRequest: 指定URL返回String类型的数据请求
2.ImageRequest: 指定URL返回图片类型的数据请求
3.JsonObjectRequest，JsonArrayReqeust(都是JsonRequest子类)。指定URL返回JSON对象或JSON数组数据的请求

1、请求图片数据

Volley为请求图片数据提供一下几个常用类。

1.ImageRequest：一个封装的图片请求，返回Bitmap对象，可以对返回的Bitmap重新定义大小。最大的优点是使用Volley的线程调度保证耗时的图片操作自动在一个工作线程中执行。
2.ImageLoader：一个帮助类，处理加载图片和缓存图片。ImageLoader是图片数据请求的统一调度，如在ListView中显示大量的图片，ImageLoader在调用普通的Volley缓存之前首先会去获取内存缓存中的图片，来防止ListView加载图片闪烁的问题。保证在获取到缓存时不会阻塞或延迟主线程。ImageLoader还合并请求响应，不用每个响应处理都为View设置一个Bitmap的处理，同时传输多个响应，提升App性能。
3.NetworkImageView：使用ImageLoader来构建图片请求，替代了ImageView来显示网络图片。如果View从视图中移除，NetworkImageView可以取消对应的网络图片的请求。

2、使用ImageRequest

下面是ImageRequest示例代码，请求网络图片并在App中显示。代码中RequestQueue使用了上面讲到的单例模式。

ImageView mImageView;
String url = "http://xxx.png";
mImageView = findViewById(R.id.myImageView);
...
// 请求网络图片，并在UI中显示
ImageRequest request = new ImageRequest(url,
    new Reponse.Listener<Bitmap>(){
        @Override
        public void onResponse(Bitmap bitmap){
            mImageView.setImageBitmap(bitmap);
        }
    },0,0,null,
    new Response.ErrorListener(){
        @Override
        public void onErrorResponse(VolleyError error) {
            mImageView.setImageResource(R.drawable.image_load_error);
        }
    });
// 使用单例模式加载图片
MySingleton.getInstance(this).addToRequestQueue(request);

4、LRU Cache例子

Volley通过DiskBasedCache提供了标准的缓存机制，用来将请求的数据缓存到磁盘上。如果使用ImageLoader，还需要实现ImageLoader.ImageCache的接口来创建内存缓存，并将缓存设为单例模式。
下面是实现内存缓存的LruBitmapCache的Java类。继承了LruCache类并且实现ImageLoader.ImageCache接口。

import android.graphics.Bitmap;
import android.support.v4.util.LruCache;
import android.util.DisplayMetrics;
import com.android.volley.toolbox.ImageLoader.ImageCache;

public class LruBitmapCache extends LruCache<String, Bitmap>
    implements ImageCache {

    public LruBitmapCache(int maxSize) {
        super(maxSize);
    }

    public LruBitmapCache(Context ctx) {
        this(getCacheSize(ctx));
    }

    @Override
    protected int sizeOf(String key, Bitmap value) {
        return value.getRowBytes() * value.getHeight();
    }

    @Override
    public Bitmap getBitmap(String url) {
        return get(url);
    }

    @Override
    public void putBitmap(String url, Bitmap bitmap) {
        put(url, bitmap);
    }

    // 根据不同屏幕大小返回缓存大小
    public static int getCacheSize(Context ctx) {
        final DisplayMetrics displayMetrics = ctx.getResources().
        getDisplayMetrics();
        final int screenWidth = displayMetrics.widthPixels;
        final int screenHeight = displayMetrics.heightPixels;
        // 4 bytes per pixel
        final int screenBytes = screenWidth * screenHeight * 4;
        return screenBytes * 3;
    }
}

使用缓存来初始化ImageLoader

RequestQueue mRequestQueue; // 假设已经初始化
ImageLoader mImageLoader = new ImageLoader(mRequestQueue, 
        new LruBitmapCache(LruBitmapCache.getCacheSize()));

五、创建自定义请求

Volley提供多种的数据请求，如JsonArrayRequest，JsonObjectRequest，StringRequest，ImageRequest等。有时我们需要除此以外的请求的返回类型数据，这是需要我们去自定义属于我们自己的数据请求Java类。创建自定义的请求类型，要考虑几点

1.继承Request<T>类，<T>泛型代表返回的数据类型。如返回是String类型的数据，创建自定义请求时候要继承Request<String>。可以参考StringRequest和ImageRequest类
2.实现方法parseNetworkResponse()和deliverResponse()

1、parseNetworkResponse

Response封装各种类型的返回数据如String，image和Json。下面是实现parseNetworkResponse()的代码范例

@Override
protected Response<T> parseNetworkResponse(
    NetworkResponse response) {
    try {
        String json = new String(response.data,
        // 获取字符编码格式
        HttpHeaderParser.parseCharset(response.headers));
        return Response.success(gson.fromJson(json, clazz),
                HttpHeaderParser.parseCacheHeaders(response));
    }       
    // 处理错误
    ...
}

注意：parseNetworkResponse()使用NetworkResponse作为它的参数。NetworkResponse包含返回响应状态码，响应的头部和比特数组类型数据。实现的方法中必须返回Response<T>类型的数据。包含你指定返回的数据对象，缓存元数据（cache metadata）或错误信息如数据解析错误；

如果你使用非标准的缓存语意，你可以自己创建Cache.Entry对象，多数情况下类似下面的返回值均是正常的。

return Response.success(myDecodedObject,
    HttpHeaderParser.parseCacheHeaders(response));

Volley在线程中调用parseNetworkResponse()方法，保证耗时操作不会阻塞和延迟主线程。

2、deliverResponse

Volley在主线程中调用parseNetworkResponse()返回的数据，大多数请求引用一个回调接口：

protected void deliverResponse(T response) {
    listener.onResponse(response);
}
3、GsonRequest请求范例

Gson是Java与JSON的Google数据解析框架。你可以定义和JSON键值一样的属性名称的Java对象，传入Gson类对象，自动为你创建Java对象。下面是Gson请求的范例：

public class GsonRequest<T> extends Request<T> {
    private final Gson gson = new Gson();
    private final Class<T> clazz;
    private final Map<String, String> headers;
    private final Listener<T> listener;

    /**
    * 创建请求，并返回JSON对象
    *
    * @param url 请求的URL
    * @param 返回的Java类
    * @param 请求头部
    */
    public GsonRequest(String url, Class<T> clazz, Map<String, String> headers,
            Listener<T> listener, ErrorListener errorListener) {
        super(Method.GET, url, errorListener);
        this.clazz = clazz;
        this.headers = headers;
        this.listener = listener;
    }

    @Override
    public Map<String, String> getHeaders() throws AuthFailureError {
        return headers != null ? headers : super.getHeaders();
    }

    @Override
    protected void deliverResponse(T response) {
        listener.onResponse(response);
    }

    @Override
    protected Response<T> parseNetworkResponse(NetworkResponse response) {
        try {
            String json = new String(response.data,
                    HttpHeaderParser.parseCharset(response.headers));
            return Response.success(gson.fromJson(json, clazz),
                    HttpHeaderParser.parseCacheHeaders(response));
        } catch (UnsupportedEncodingException e) {
            return Response.error(new ParseError(e));
        } catch (JsonSyntaxException e) {
            return Response.error(new ParseError(e));
        }
    }
}
