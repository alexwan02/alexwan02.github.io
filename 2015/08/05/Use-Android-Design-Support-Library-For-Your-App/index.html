Use Android Design Support Library For Your App

发表于 2015-08-05   |  


作为一个Android开发人员，大家都或多或少地了解了Android官方推出的Material Design依赖库。在官方推出依赖库之前，我们要是想使用Material Design的UI风格，只能使用第三方的开源依赖库。但是现在Google在2015年的I/O的开发者大会上宣布了名为Android Design Support Library的官方的Material Design UI 风格的依赖库。在本篇文章我们逐步来介绍如何来使用这个库.
好了，废话不多说，进入主题！

OK 开始吧！

一、从Github上把源码Clone下来

我已经把项目的基本配置资源文件上传到了Github上。你可以Clone到本地导入到AS中。代码中MainActivity是完成后的代码，你可以在CodeLabActivity 类中根据我们的步骤来体验Material Design设计的快感。

注：推荐使用Google的官方IDE Android Studio，本篇也是以AS来介绍

你需要做的第一步是成功地把Demo运行起来，简单吧！

二、添加Android Design Support Library Dependency

如何添加Android Design Support Library到我们的项目呢，也是很简单的。只需要在项目的build.gradle配置文件中添加上那么一句话

compile 'com.android.support:design:22.2.1'

Very Good

注:Design Support Library依赖Support v4 and AppCompat v7，想要使用Design Support Library要记住还要导入v4和v7的依赖哟。
另外，从Github上Clone的代码已经添加了上面一行配置代码，如果你是自己创建的新项目，需要自己添加这行配置。

三、添加FAB

FAB（Floating Action Button简称）是一个带有阴影效果的圆形Button，典型的Meterail Design设计风格。所以我们从添加FAB开始，在布局文件中添加FloatingActionButton，将它放置在FrameLayout的布局里面是因为FAB需要一个父布局来使它位于屏幕的右下角。把下面的代码替换掉activity_code_lab.xml布局文件中的原有代码

<android.support.v4.widget.DrawerLayout ...
xmlns:app="http://schemas.android.com/apk/res-auto"
....>

<FrameLayout
    android:id="@+id/rootLayout"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    >

    <android.support.design.widget.FloatingActionButton
        android:id="@+id/fabBtn"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:layout_gravity="bottom|right"
        android:src="@drawable/ic_plus"
        app:fabSize="normal" />

</FrameLayout>

...

</android.support.v4.widget.DrawerLayout>

android:src ：用来定义你需要ICON(建议为40dp大小的透明png文件)

android:fabSize ：用来定义FAB的大小

normal：使用标准的56dp大小的button
mini：使用40dp大小的button

四、Play with Snackbar

Snackbar, 一个显示在屏幕下方的黑色消息框。SnackBar类似与Toast，但是不同于Toast是悬浮显示在屏幕上，而SnackBar而是UI的一部分。



Snackbar使用的方式也类似Toast

Snackbar.make(someView," Hello,I am Snackbar!", Snackbar.LENGTH_SHORT).setAction("Undo", new View.onClickListener(){

        }

setAction()：定义显示在SnackBar右边的按钮的Action（可选）

六、放弃ActionBar，使用Toolbar

Toolbar 并不是Android Design Support Library的一部分，但是需要配合使用。
Toolbar相比ActionBar更加灵活，和Android Design Support库使用具备ActionBar没有的新特性。
而转换使用Toolbar也很简单，只需要在定义Activity的样式xml文件中添加几句话即可隐藏掉ActionBar。

    <item name=“windowActionBar”>false</item>
    <item name=“windowNoTitle”>true</item>
</style>

在CoordinatorLayout中使用Toolbar组件，要位于最上。

<android.support.design.widget.CoordinatorLayout 
    …>
    <android.support.v7.widget.Toolbar
        androi:id=“@+id/toolBar”
        android:layout_width=“match_parent”
        android:layout_height="?attr/actionBarSize"
        android:background="?attr/colorPrimary"
        app:popupTheme="@style/ThemeOverlay.AppCompat.Light"
        app:theme="@style/ThemeOverlay.AppCompat.Dark.ActionBar"
    />
    <android.support.design.widget.FloatingActionButton
        ...>
    </android.support.design.widget.FloatingActionBar>
</android.support.deign.widget.CoordinatorLayout>

在Java代码中使用Toolbar替代ActionBar

Toolbar toolbar;

private void initInstance(){
    toolbar = (Toolbar) findViewById(R.id.toolbar);
    setSupportActionBar(toolbar);
    ...
}

运行代码，没有问题。但是前面我们说在CoordinatorLayout中的布局是互相协调的，不会和之外的布局协调
而Toolbar是位于整个xml布局文件顶部，需要和所有的View相适配。怎么办呢？因此我们引入AppBarLayout
，这个布局是专为了Toolbar更好的和CoordinatorLayout运作。

    <android.support.design.widget.FloatingActionButton
        ...>
    </android.support.design.widget.FloatingActionButton>

八、使用TabLayout

我们经常会在APP中使用TAB，来增加切换的效果。在Android Meterial Design 依赖库之前我们一般使用第
三方依赖库来实现Meterial Design的效果。而现在可以直接使用官方依赖包提供的TabLayout实现相同的功能。
TabLayout与Toolbar一样要放置在AppBarLayout中，因为AppBarLayout继承了LinearLayout，所
以直接放在ToolBar下方：

<android.support.design.widget.AppBarLayout ...>

    <android.support.v7.widget.Toolbar ... />

    <android.support.design.widget.TabLayout
        android:id="@+id/tabLayout"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"/>

</android.support.design.widget.AppBarLayout>

在Java代码中添加Tab按钮

TabLayout tabLayout

private void initInstance(){
    tabLayout = (TabLayout) findViewById(R.id.tabLayout);
    tabLayout.addTab(tabLayout.newTab.setText(Tab 1));
    tabLayout.addTab(tabLayout.newTab.setText(Tab 2));
    tabLayout.addTab(tabLayout.newTab.setText(Tab 3));
    ...
}

上效果



Tab的背景颜色自动设置为 Primary Color 。我们注意到Tab字体颜色是黑色的，在布局文件中指定TabLayout
的主题样式可以给字体颜色换色。

<android.support.design.widget.TabLayout
    ...
    app:theme="@style/ThemeOverlay.AppCompat.Dark.ActionBar"/>

字体变成白色了



使用TabLayout经常会和ViewPager来配合使用，TabLayout提供方法setupWithViewPager(...)设置
TabLayout用到的ViewPager。
TabLayout有两个属性我们可以用来调整TabLayout中Tab的布局效果

* app:tabMode - fixed: 固定tab的位置，当tab数量过多时，超出屏幕范围后。无法显示更多。
                scrollable: tab数量超出屏幕范围，可以滚动tab，显示更多的tab
* app:tabGravity - fill: tab自动填充tabLayout
                    center: tab位于TabLayout的中间

如果tabMode为scrollable模式时候，tabGravity的属性自动忽略

只有Toolbar滑动出屏幕了，而TabLayout仍然显示，是因为我们没有在TabLayout中设置Scroll Flags属性

<android.support.design.widget.TabLayout
    ...
    app:layout_scrollFlags="scroll|enterAlways" />

Scroll Flags属性
scroll - view随着屏幕滚动
enterAlwaysCollapsed - 属性定义了什么时候View返回屏幕。如果View设置了最小高度，View滑出的时
候只显示最小高度大小的View，当Content内容全部滑出显示的时候，View的剩下部分才全部显示

enterAlways - 任何手势下滑操作都会让View可见

exitUntilCollapsed - 退出到最小高度。这种模式最常用。

<android.support.v7.widget.Toolbar
    ...
    android:layout_height="192dp"
    android:gravity="bottom"
    android:paddingBottom="12dp"
    android:minHeight="?attr/actionBarSize"
    app:layout_scrollFlags="scroll|exitUntilCollapsed"
    />

十一、Toolbar的收起展开

在使用exitUntilCollapsed的时候，Toolbar的图标消失了，在App中希望始终显示导航的图标。
Design Support Library为Toolbar提供的新的布局CollapsingToolbarLayout可以让Toolbar的
图标在收起的时候不被隐藏
添加CollapsingToolbarLayout只需要下面几步
1.将Toolbar放在CollapsingToolbarLayout中。
2.移除Toolbar的layout_scrollFlags属性
3.CollapsingToolbarLayout添加layout_scrollFlags属性，设置为scroll|exitUntilCollapsed
4.改变AppBarLayout展开的高度。在例子中设置为256dp大小的高度

<android.support.design.widget.AppBarLayout
    android:layout_width="match_parent"
    android:layout_height="256dp">
    <android.support.design.widget.CollapsingToolbarLayout
        android:id="@+id/collapsingToolbarLayout"
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        app:layout_scrollFlags="scroll|exitUntilCollapsed">

        <android.support.v7.widget.Toolbar
            android:id="@+id/toolbar"
            android:layout_width="match_parent"
            android:layout_height="?attr/actionBarSize"
            android:background="?attr/colorPrimary"
            android:minHeight="?attr/actionBarSize"
            app:theme="@style/ThemeOverlay.AppCompat.Dark.ActionBar"
            app:popupTheme="@style/ThemeOverlay.AppCompat.Light" />

    </android.support.design.widget.CollapsingToolbarLayout>

</android.support.design.widget.AppBarLayout>

效果

Toolbar的图标仍然滑出的了屏幕。我们要让它固定显示在屏幕的顶部

<android.support.v7.widget.Toolbar

    app:layout_collapseMode="pin"
/>

总结

Android Design Support Library的还是不错的，非常推荐在你的App使用这个依赖包。现在还有一些
bug，因此在使用的时候需要注意这些问题。

Updated [24/07/2015]: From v22.2.0 to v22.2.1
在版本v22.2.1中修复了一些问题

  • TabLayout 更加完美运行在AppBarLayout中，运行更加流畅
  • FloatingActionButton 在Android 5.0版本以上不需要额外设置app:borderWidth

  • exitUntilCollapsed 运行时与描述更相符合了

仍然存在的Bug

  • FloatingActionButton 的margin属性在Android 5.0以上版本依然与之前的版本不同。需要额外设置margin值
    新问题
  • FloatingActionButton 在Android 5.0以下，当FAB放在CoordinatorLayout布局中时候，margin变为0.但是在其他的布局中有一些margin值
