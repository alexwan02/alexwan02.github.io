Use Android Design Support Library For Your App

发表于 2015-08-05   |  


作为一个Android开发人员，大家都或多或少地了解了Android官方推出的Material Design依赖库。在官方推出依赖库之前，我们要是想使用Material Design的UI风格，只能使用第三方的开源依赖库。但是现在Google在2015年的I/O的开发者大会上宣布了名为Android Design Support Library的官方的Material Design UI 风格的依赖库。在本篇文章我们逐步来介绍如何来使用这个库.
好了，废话不多说，进入主题！

OK 开始吧！

一、从Github上把源码Clone下来

我已经把项目的基本配置资源文件上传到了Github上。你可以Clone到本地导入到AS中。代码中MainActivity是完成后的代码，你可以在CodeLabActivity 类中根据我们的步骤来体验Material Design设计的快感。

注：推荐使用Google的官方IDE Android Studio，本篇也是以AS来介绍

你需要做的第一步是成功地把Demo运行起来，简单吧！

Activity的theme的风格调整，在AppTheme中添加Material Design的样式

二、添加Android Design Support Library Dependency

如何添加Android Design Support Library到我们的项目呢，也是很简单的。只需要在项目的build.gradle配置文件中添加上那么一句话

compile 'com.android.support:design:22.2.1'

Very Good

注:Design Support Library依赖Support v4 and AppCompat v7，想要使用Design Support Library要记住还要导入v4和v7的依赖哟。
另外，从Github上Clone的代码已经添加了上面一行配置代码，如果你是自己创建的新项目，需要自己添加这行配置。

三、添加FAB

FAB（Floating Action Button简称）是一个带有阴影效果的圆形Button，典型的Meterail Design设计风格。所以我们从添加FAB开始，在布局文件中添加FloatingActionButton，将它放置在FrameLayout的布局里面是因为FAB需要一个父布局来使它位于屏幕的右下角。把下面的代码替换掉activity_code_lab.xml布局文件中的原有代码

<android.support.v4.widget.DrawerLayout ...
xmlns:app="http://schemas.android.com/apk/res-auto"
....>

<FrameLayout
    android:id="@+id/rootLayout"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    >

    <android.support.design.widget.FloatingActionButton
        android:id="@+id/fabBtn"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:layout_gravity="bottom|right"
        android:src="@drawable/ic_plus"
        app:fabSize="normal" />

</FrameLayout>

...

</android.support.v4.widget.DrawerLayout>

android:src ：用来定义你需要ICON(建议为40dp大小的透明png文件)

android:fabSize ：用来定义FAB的大小

normal：使用标准的56dp大小的button
mini：使用40dp大小的button

下图就是我们运行代码后的结果啦！



但是我们运行在Android 5.0上时候会有问题，看下图



为什么有这个问题，其实这是个BUG，相信未来Design Library的开发者会修复的。但是我又想现在就使用怎么办，也是有解决的办法的。我们在dimen文件中配置一下就可以了，在API Level 21+ 以上dimens文件中设置FAB’s margin right and margin bottom为16dp，以下的版本设置为0dp就能够解决问题了。

res/values/dimens.xml

<dimen name="codelab_fab_margin_right">0dp</dimen>
<dimen name="codelab_fab_margin_bottom">0dp</dimen>

res/values-v21/dimens.xml

<dimen name="codelab_fab_margin_right">16dp</dimen>
<dimen name="codelab_fab_margin_bottom">16dp</dimen>

res/layout/activity_code_lab.xml

<android.support.design.widget.FloatingActionButton
    ...
    android:layout_marginBottom="@dimen/codelab_fab_margin_bottom"
    android:layout_marginRight="@dimen/codelab_fab_margin_right"
.../>

哈哈 大功告成！



FAB默认的状态：空闲时：6dp、Pressed 状态：12dp。你也可以使用app:elevation 设置空闲时的shadow depth，使用app:pressedTranslationZ 设置Pressed状态的shadow depth

FAB使用了强调色，可以使用属性app:backgroundTint设置FAB的颜色。
像一般的Button一样，我们也可以使用setOnclickListener设置FAB的点击事件。

四、Play with Snackbar

Snackbar, 一个显示在屏幕下方的黑色消息框。SnackBar类似与Toast，但是不同于Toast是悬浮显示在屏幕上，而SnackBar而是UI的一部分。




Snackbar使用的方式也类似Toast

Snackbar.make(someView,”Hello . I am Snackbar !”, Snackbar.LENGTH_SHORT)
.setAction(“Undo”, new View.onClickListener(){
@Override
public void onClick(View v){

}
})
.show()；

make()：第一个参数表示显示位于View或Layout的bottom位置。在例子中使用FrameLayout
setAction()：定义显示在SnackBar右边的按钮的Action（可选）
接下来我们添加以下代码

FrameLayout rootLayout;

fabBtn = (FloatingActionButton) findViewById(R.id.fabBtn);

点击看看效果


显示出来了，但是把FAB遮住了，是因为FAB与Snackbar之间没有布局没有关联。因此我们引入一个特殊的新布局CoordinatorLayout 让子View互相适配。

五、使用CoordinatorLayout

修改布局文件，替换掉原来的FrameLayout
res/layout/activity_code_lab.xml

<android.support.design.widget.CoordinatorLayout
    android:id="@+id/rootLayout"
    android:layout:width="match_parent"
    android:layout:height="match_parent">
    <android.support.design.widget.FloatingActionButton.../>
</android.support.design.widget.CoordinatorLayout>

修改CodeLabActivity.java的rootLayout 的类型为CoordinatorLayout

// FrameLayout rootLayout;
CoordinatorLayout rootLayout;

//rootLayout = (FrameLayout) findViewById(R.id.rootLayout);
rootLayout = (CoordinatorLayout) findViewById(R.id.rootLayout);

使用CoordinatorLayout在Android 4.4上有个bug，FAB的margin会变为0.




这个BUG值存在Android Design Support Library 22.2.2 ，等待Google修复。现在使用必须设置 FAB的margin right 和 margin bottom的值为16dp。

res/values/dimens.xml

16dp
16dp

看看效果

六、放弃ActionBar，使用Toolbar

Toolbar 并不是Android Design Support Library的一部分，但是需要配合使用。
Toolbar相比ActionBar更加灵活，和Android Design Support库使用具备ActionBar没有的新特性。
而转换使用Toolbar也很简单，只需要在定义Activity的样式xml文件中添加几句话即可隐藏掉ActionBar。

<style name=“AppTheme” parent=“Theme.AppCompat.Light.DarkActionBar”>
    <item name=“windowActionBar”>false</item>
    <item name=“windowNoTitle”>true</item>
</style>

在CoordinatorLayout中使用Toolbar组件，要位于最上。

<android.support.design.widget.CoordinatorLayout 
    …>
    <android.support.v7.widget.Toolbar
        androi:id=“@+id/toolBar”
        android:layout_width=“match_parent”
        android:layout_height="?attr/actionBarSize"
        android:background="?attr/colorPrimary"
        app:popupTheme="@style/ThemeOverlay.AppCompat.Light"
        app:theme="@style/ThemeOverlay.AppCompat.Dark.ActionBar"
    />
    <android.support.design.widget.FloatingActionButton
        ...>
    </android.support.design.widget.FloatingActionBar>
</android.support.deign.widget.CoordinatorLayout>

在Java代码中使用Toolbar替代ActionBar

Toolbar toolbar;

private void initInstance(){
    toolbar = (Toolbar) findViewById(R.id.toolbar);
    setSupportActionBar(toolbar);
    ...
}

运行代码，没有问题。但是前面我们说在CoordinatorLayout中的布局是互相协调的，不会和之外的布局协调
而Toolbar是位于整个xml布局文件顶部，需要和所有的View相适配。怎么办呢？因此我们引入AppBarLayout
，这个布局是专为了Toolbar更好的和CoordinatorLayout运作。

<android.support.design.widget.CoordinatorLayout
    ...>

    <android.support.design.widget.AppBarLayout
        android:layout_width="match_parent"
        android:layout_height="wrap_content">
        <android.support.v7.widget.Toolbar
            .../>
    </android.support.design.widget.AppBarLayout>

    <android.support.design.widget.FloatingActionButton
        ...>
    </android.support.design.widget.FloatingActionButton>
</android.support.design.widget.CoordinatorLayout>

运行代码测试一下，完美！
