Android ：ListView异步加载图片的优化

发表于 2015-09-12   |  


在Android开发中我们会选择ListView来加载列表数据。新手容易因为不注意或者不了解ListView加载Adapter机制而出现这些错误。

  1. 如何重用convertView
  2. 如何使用ViewHolder重用ItemView
  3. 如何解决异步加载数据时候导致内容错位闪烁等问题

使用ListView时应该需要注意这些问题。下面我们结合ListView和Adapter的源码来分析其内部是如何实现以及如何去解决这三个问题。

一、ListView重用View原理

1.源码分析

ListView调用setAdapter(Adapter adapter)方法加载数据。初始时根据屏幕给ListView的高度和每个Adapter的高度计算一次可以加载的Adapter的数量，定义要重用的Item的。每个Item在进入屏幕显示时都会调用Adapter中的方法getView(int postion , View converView , ViewGroup group)，在ListView上下滑动时，如果一个Item滚出屏幕，另外一个Item会进入屏幕，由getView()判断进入屏幕的Item是否重用滚出屏幕的item的视图。那么ListView在上下滑动的时候是如何调用adapter的getView方法的呢？接下来我们分析源码实现。

ListView在上下滑动的时候会去调用makeAndAddView()方法来获取新的View，这个新的View可能是创建新的，也可能是重用之前创建好的View.

2.解决重用问题

ListView在初始化的时候，根据屏幕大小和Adapter中View大小来创建Adapter的View个数。之后再去判断是否复用创建好的View。一般我们会在Adapter方法getView中使用如下的代码去判断当前View是否已经存在了，来解决重用convertView的问题。使用ViewHolder来解决重用ViewHolder的子View的问题

二、ListView异步加载网络数据优化

ListView在上下滑动的时候重用缓存的View，如果Item中异步加载网络数据的时候，因为请求网络数据的延迟性，还在加载网络数据的时候，View已经被重用（如快速上滑下滑），且数据在滑动过程加载完毕，导致重用的View仍然显示之前的数据，造成显示重复和错乱的情况。在加载网络图片的时候，有时因为加载重用View的原因。导致加载之前异步的图片后，当前图片加载完毕后又变成当前加载图片，造成图片闪烁的问题。
所以在异步加载数据的时候，我们设置一个tag属性来判断当前加载的数据是否与异步加载完成时的tag属性是否一致，一致则显示，不一致则设置默认值。

    ViewHolder holder;

        convertView.setTag(holder);

    imageView.setImageDrawable(new ColorDrawable);
    // 加载URL
    ...
    return convertView;
}
